Backward Pass

In the forward pass we calculate the label estimates \mathbf{\hat{y}} undefined based on the features \mathbf{X} undefined. These estimates allow us to measure the loss L undefined, like the cross-entropy loss for classification or the mean squared error for regression.

In the backward pass we calculate the gradients and apply gradient descent.

\begin{aligned} \mathbf{w}_{t+1} & \coloneqq \mathbf{w}_t - \alpha \mathbf{\nabla}_w \\ \mathbf{b}_{t+1} & \coloneqq \mathbf{b}_t - \alpha \mathbf{\nabla_b}\\ \end{aligned} undefined

This procedure should look familiar, because in essence these are the same steps, that we used in linear and logistic regression, but how exactly should we calculate those gradients? When we define gradient descent as \mathbf{w}_{t+1} \coloneqq \mathbf{w}_t - \alpha \mathbf{\nabla} undefined, we imply that the vector \mathbf{w} undefined encompasses all the weights from all the layers in the neural network. Consider for example the forward pass from the previous section.

\mathbf{\hat{y}} = \sigma( \sigma(\sigma(\mathbf{X} \mathbf{W}^{<1>T}) \mathbf{W}^{<2>T})\mathbf{W}^{<3>T}) undefined

Just two small hidden layers and an output layer create a complex nested expression and yet we are expected to exactly calculate all the derivatives of the form \dfrac{\partial}{\partial w^{<l>}_{k,t}} L undefined and \dfrac{\partial}{\partial b^{<l>}_{k}} L undefined, where the superscript <l> undefined is the layer in the neural network, k undefined is the row in the weight matrix and t undefined is the column in the weight matrix. In other words we need to figure out how individual weights and biases impact the loss in order to adjust those weights and biases to reduce the loss. That is exactly what the backpropagation algorithm is used for.

The backpropagation algorithm calculates the gradients of the loss with respect to weights and biases for each layer and neuron of the neural network. Gradient descent on the other hand utilizes those gradients to adjust the weights and biases to reduce the loss.

The backpropagation algorithm makes extensive use of the chain rule, which allows us to find derivatives of composite functions of the form f(x) = h(g(x)) undefined.

The chain rule states that we can find the derivative of f(x) undefined with respect to x undefined by separately finding the derivatives \dfrac{dh}{dg} undefined and \dfrac{dg}{dx} undefined calculating the product of both derivatives.

\dfrac{d}{dx}f(x) = \dfrac{dh}{dg} \dfrac{dg}{dx} \\ undefined

Intuitively this makes sense because the terms cancel out and we are left with our desired derivative.

\dfrac{d}{dx}f(x) = \dfrac{dh}{dg} \dfrac{dg}{dx} = \dfrac{dh}{\cancel{dg}} \dfrac{\cancel{dg}}{dx} = \dfrac{dh}{dx} \\ undefined

Let us work through a simple example to make the intuition stick. We assume we deal with f(x) = (5x+100)^2 undefined. The function f(x) undefined is actually a composite of the function h(g) = g^2 undefined and g(x) = 5x + 100 undefined.

First we find the derivatives \dfrac{dh}{dg} = 2g = 2(5x+100) undefined and \dfrac{dg}{dx} = 5 undefined. Lastly we apply the chain rule by multiplying the derivatives to end up with \dfrac{d}{dx}f(x) = 10(5x+100) = 50x + 1000 undefined. We can easily verify that this is the correct derivative by calculating the derivative directly after applying the binomial formula.

\begin{aligned} \dfrac{d}{dx}f(x) & = \dfrac{d}{dx} (5x + 100)^2 \\ & = \dfrac{d}{dx}25x^2 + 1000x + 1000 \\ & = 50x + 1000 \end{aligned} undefined

While we used a composition of two functions for the ease of explanations, the chain rule is not restricted to a composition of just two functions. For example given a composite function of the following form f(x) = h(g(u(v(x)))) undefined the derivative can be calculated as follows using the chain rule: \dfrac{d}{dx}f(x) = \dfrac{dh}{dg} \dfrac{dg}{du} \dfrac{du}{dv} \dfrac{dv}{dx} undefined.

The chain rule is also not restricted to functions with just one variable. Let us for example assume that we use two single variable functions x(t) undefined and y(t) undefined. Let us further assume that we calculate the function f(x,y) undefined. The derivative of f(x,y) undefined with respect to t undefined can be calculated using the multivariable chain rule, where \dfrac{d}{dt}f(x, y) = \dfrac{df}{dx} \dfrac{dx}{dt} + \dfrac{df}{dy} \dfrac{dy}{dt} undefined.

When we look at the neural network from the previous section, it becomes clear, that the network is a composion of many functions with many variables.

\mathbf{\hat{y}} = \sigma( \sigma(\sigma(\mathbf{X} \mathbf{W}^{<1>T}) \mathbf{W}^{<2>T})\mathbf{W}^{<3>T}) undefined

That means that we should be able to calculate the partial derivative of the loss function L undefined with respect to any of the weights \mathbf{w} undefined or biases \mathbf{b} undefined using the chain rule.

But backpropagation is more than just the chain rule, it is the chain rule combined with an efficient calculation algorithm.

To understand why we need an efficient algorithm, let us use the interactive example below. You can click on any of the weights to mark the nodes and paths that are necessary to calculate the derivative for that particular weight.

The weights that connect the inputs (leftmost layer) and the second layer have an impact on all neurons in the third layer. This is essentially the reason why we need to use the multivariable chain rule.

Neurons are influenced by many different weights from previous layers. For example all the weights that connect the first and the second layer, have an impact on the first activation in the third layer and half the weights that connect the second and the third layer have an impact on the first activation in the third layer. This implies that when we apply the chain rule, gradients of several weights will have many of the same components in the product of the chain rule. To avoid duplicate calculations, some of the results will be saved in the backward pass. Backpropagation calculates the gradients for the weights of the last layer first. Many of the intermediate results are required parts for the calculation of gradients for of all the previous layers. The calculations of the gradients in the second to the last layers are also saved, as those are required parts for all the layers that precede the second to last layer. The process can be applied indefinetely, no matter how many layers the neural network has.

It makes sense to work through at least one example to understand how the backpropagation algorithm works. For that we are going to use a simple neural network. The network has two features as inputs, two hidden layers with two neurons each and the output layer has a single neuron.

In this example we utilize the cross-entropy, but the similar derivations can be made with the mean squared error loss.

L = -\dfrac{1}{n}\sum_i \big[y^{(i)}\ln \hat{y}^{(i)} + (1-y^{(i)})\ln (1 - \hat{y}^{(i)})\big] undefined

While we always work with a batch or mini batch in practice, for the sake of simplicity we are going to focus on a single training example. This simplifies the explanations and the notation significantly. Just keep in mind that in practice we would make the calculations for a whole batch and calculate the mean of the gradients, before we take a gradient descent step.

L = - y\ln \hat{y} -(1-y)\ln(1 - \hat{y}) undefined

Additionally we replace the prediction output \hat{y} undefined by the output of the last layer a^{<3>} undefined. This is a notational change, that has no impact on the calculation.

L = - y\ln a^{<3>}-(1-y)\ln(1 - a^{<3>}) undefined

We are going to apply the same strategy that we used in linear and logistic regression. We keep taking partial derivatives with respect to some intermediary values and apply the chain rule once we have all the necessary components.

First we calculate the derivative of the loss L undefined with respect to the last activation. We utilize the fact that the derivative of the natural log is \dfrac{d}{dx}lnx = \dfrac{1}{x} undefined and end up with.

\dfrac{d}{d a^{<3>}}L = -y\dfrac{1}{a^{<3>}} + (1- y)\dfrac{1}{1-a^{<3>}} undefined

In the next step we calculate the derivative \dfrac{da^{<3>}}{dz^{<3>}} undefined. We use the sigmoid as our activation function \sigma(z) = \dfrac{1}{1+e^{-z}} undefined, which means that the derivative is \dfrac{da^{<3>}}{dz^{<3>}} = a^{<3>}(1-a^{<3>}) undefined.

With those two derivatives in hand we can calculate an intermediary error \delta^{<3>} undefined. The error is simply the application of the chain rule, therefore \delta^{<3>} = \dfrac{dL}{da^{<3>}} \dfrac{da^{<3>}}{dz^{<3>}} undefined. The deltas \delta undefined are the parts of the calculation that are going to be used for the gradients in all the previous layers and the current layer. The deltas are always partial derivatives with respect to the net inputs z undefined of that particular layer.

Once we have the \delta undefined for the last layer, we can reuse this value to calculate the gradient with respect to the weights that connect the second to last and the last layers.

\dfrac{dL}{dw^{<3>}_{1,1}} = \dfrac{dL}{da^{<3>}}\dfrac{da^{<3>}}{dz^{<3>}} \dfrac{dz^{<3>}}{dw_{1,1}^{<3>}} = \delta^{<3>}\dfrac{dz^{<3>}}{dw_{1,1}^{<3>}} undefined

\dfrac{dL}{db^{<3>}} = \dfrac{dL}{da^{<3>}}\dfrac{da^{<3>}}{dz^{<3>}} \dfrac{dz^{<3>}}{db^{<3>}} = \delta^{<3>}\dfrac{dz^{<3>}}{db^{<3>}} undefined

The net input in the third layer is represented as z^{<3>}_1 = a^{<2>}_1 w_{11}^{<3>} + a^{<2>}_{12} w_{12}^{<3>} + b_1^{<3>} undefined. Calculating the partial derivative of the net input with respect to weights and biases shoud not pose any problems, as this is essentially the same exercise we did with linear regression: \dfrac{\partial z^{<3>}_1}{\partial w_{1,1}^{<3>}} = a^{<2>}_1 undefined , \dfrac{\partial z^{<3>}_1}{\partial w_{1,2}^{<3>}} = a^{<2>}_2 undefined, \dfrac{\partial z^{<3>}_1}{\partial b_1^{<3>}} = 1 undefined.

After the calculation of the gradients for the weights and biases of the last layer, we can move one step backwards. Once again our goal is to calculate the deltas \delta undefined, but this time we can reuse the \delta undefined from the next layer. Let us remember that the deltas \delta undefined are derivatives with respect to the net inputs z undefined. Therefore there are as many deltas as there are neurons in a layer. In the second to last layer there are two neurons, so we have to calculate two deltas.

\delta^{<2>}_1 = \delta^{<3>} \dfrac{dz^{<3>}}{da^{<2>}_1} \dfrac{da^{<2>}_1}{dz_1^{<2>}} undefined

\delta^{<2>}_2 = \delta^{<3>} \dfrac{dz^{<3>}}{da^{<2>}_2} \dfrac{da^{<2>}_2}{dz_2^{<2>}} undefined

This time around we need calculate the derivatives of net input z^{<3>}_1 = a^{<2>}_1 w_{11}^{<3>} + a^{<2>}_{12} w_{12}^{<3>} + b_1^{<3>} undefined with respect to the activations a undefined, which turn out to be the corresponding weights: \dfrac{\partial z^{<3>}_1}{\partial a^{<2>}_1 } = w_{1,1}^{<3>} undefined and \dfrac{\partial z^{<3>}_1}{\partial a^{<2>}_2} = w_{1,2}^{<3>} undefined.

At this point we used the delta \delta^{<3>} undefined five times. The more layers and neurons we have, the more computational resources we save by reusing the deltas. This is the magic of backpropagation.

We continue with our path applying the chain rule and calculate the gradients with respect to weights and biases in the second layer.

\dfrac{dL}{dw^{<2>}_{1,1}} = \delta_1^{<2>} \dfrac{dz^{<2>}_1}{dw^{<2>}_{1,1}} undefined

\dfrac{dL}{dw^{<2>}_{1,2}} = \delta_1^{<2>} \dfrac{dz^{<2>}_1}{dw^{<2>}_{1,2}} undefined

\dfrac{dL}{db^{<2>}_{1}} = \delta_1^{<2>} undefined

\dfrac{dL}{dw^{<2>}_{2,1}} = \delta_2^{<2>} \dfrac{dz^{<2>}_2}{dw^{<2>}_{2,1}} undefined

\dfrac{dL}{dw^{<2>}_{2,2}} = \delta_2^{<2>} \dfrac{dz^{<2>}_2}{dw^{<2>}_{2,2}} undefined

\dfrac{dL}{db^{<2>}_{2}} = \delta_2^{<2>} undefined

As you might imagine this pattern of computing and reusing deltas can theoretically continue indefinetely. No matter how many layers the neural network has, we calculate the deltas based on the deltas from the next layers and thus avoid inefficient recalculations.

When we deal with several hidden layers and/or several outputs, we inevitably encounter the multivariable chain rule in the calculation of the \delta undefined. This is due to the fact, that some of the activations and thus net inputs are used as the inputs for all the neurons in the next layer.

\delta^{<1>}_1 = \delta^{<2>}_1 \dfrac{dz^{<2>}_1}{da^{<1>}_1} \dfrac{da^{<1>}_1}{dz_1^{<1>}} + \delta^{<2>}_2 \dfrac{dz^{<2>}_2}{da^{<1>}_1} \dfrac{da^{<1>}_1}{dz_1^{<1>}} undefined

\delta^{<1>}_2 = \delta^{<2>}_1 \dfrac{dz^{<2>}_1}{da^{<1>}_2} \dfrac{da^{<1>}_2}{dz_2^{<1>}} + \delta^{<2>}_2 \dfrac{dz^{<2>}_2}{da^{<1>}_2} \dfrac{da^{<1>}_2}{dz_2^{<1>}} undefined

Before you move to that example, we have to warn you that you are not dealing with the most efficient implementation. For once we use batch and not mini batch gradient descent. That can slow down training considerably, because in mini batch gradient descent the algorithm takes many gradient descent steps in one epoch and thus moves towards the optimal value several times in an epoch, while in simple batch gradient descent only one optimization step is taken during an epoch. Additionally we use the sigmoid activation function, which is not considered to be state of the art any more. We want to focus on the most straightforward implementation for the time being and introduce improvements in a dedicated chapter.

Usually 10000 steps are sufficient to find weights for a good decision boundary. Try to observe how the cross-entropy and the shape of the decision boundary change over time. At a certain point you will most likely see a sharp drop in cross entropy, this is when things will start to improve greatly.
