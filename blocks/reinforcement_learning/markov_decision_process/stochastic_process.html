MDP as Stochastic Process

The name Markov decison process was not created out of thin air. It is categorized by three distinct parts. It is a stochastic process, it has the Markov property and it provides a mechanism for the agent to make decisions. In the following sections we will unpack and discuss the three parts.

Stochastic Process

A stochastic process is defined as a sequence of random variables.

In the animation above we see the result of an unlimited number of random coin tosses, where each toss results either in an H, the head of a coin, or a T, the tail of a coin. In this example we deal with a fair coin, that means that the probability of drawing eather heads or tails is always exaclty 50%. What we have described is the so called Bernoulli process. The process evolves randomly by continuosly flipping an imaginary fair coin.

For our purposes it is more convenient to represent heads and tails as a 0's and 1's. We can look at these numbers as states in an enviroment, where the environment transitions from one state into an another with a 50% probability and stays in the same state with 50% probability.

For the Bernoulli process we can state the following: Pr(S_{t+1} \mid S_t) = Pr(S_{t+1}) undefined. Pr(S_{t+1}) undefined is the probability that a certain state will be tossed, in the above case Pr(S_{t+1}=0) = 0.5 undefined and Pr(S_{t+1}=1)=0.5 undefined . Pr(S_{t+1} \mid S_t) undefined, reads as S_{t+1} undefined given S_t undefined, depicts a conditional probability, where the probability of being in the new state S_{t+1} undefined depends on the current state S_t undefined. For example Pr(S_{t+1}=0|S_t=1) undefined shows the probability of a coin toss having a value of HEADS when the previous toss had a value of TAILS. When you consider a coin toss, then the new occurrence of either heads or tails does not depend on the previous specific value of the toss. The events are independent. Pr(S_{t+1} \mid S_t) = Pr(S_t) undefined means that knowing the last value of a coin toss does not give us any more knowledge regarding the future toss and therefore Pr(S_{t+1} \mid S_t) = Pr(S_t) = 0.5 undefined.

If we used the Bernoulli process in reinforcement learning, our agent would drift from one state into the next state, without any agency and any rewards to guide his actions.

Markov Chain

A Markov chain is a stochastic process that has the Markov property.

The above animation depicts a simple, two state, Markov chain. Unlike in the case of a Bernoulli process the next state is not independent of the current state. The probability of the next state depends on the current state. If the environment is in the 0's state the probability to remain is 20% and the probability to transition into the 1's state is 80%. If the environment is in the 1's state on the other hand, there is a 50/50 chance of either staying in the same state or transitioning into the 0's state.

The Markov chain exhibits a so called Markov property. In simple words that means that the probability of transitioning into the next state is only dependent on the current state. The whole history of past states is irrelevant. This property is sometimes called memorylessness.

Mathematically the Markov property can be depicted as follows.

Pr[S_{t+1} \mid S_t] = Pr[S_{t+1} \mid S_1, .... , S_t] undefined

Markov Reward Process

The Markov reward process adds a reward for to each state. Whenever a time step passes a reward is generated by a Markov reward process. The reward can theoretically also be calculated stochastically, but for simplicity sake let us assume a deterministic reward.

In a Markov reward process the sequence of random variables not only contains states (reddish tint) but also rewards (white tint). The process drifts randomly from one state into the next and gives out rewards, but the agent has no influence on the process whatsoever.

Markov Decision Process

A Markov decision process adds, as the name suggests, decisions to the Markov chain. In other words the agent gets some agency over it's decision.

A Markov decision process is a Markov chain, where a part of the chain is influenced by the decisions of the agent.

In the example above the environment has still two states, state 0 and state 1 (represented by red boxes). Landing in 0's state is more advantagous as the agent receives a reward of 5, while landing in the 1's state generates a reward of -1 (white circles). In both states the agent can choose between two actions (blue circles). The 0's action is the action toward the 0's state and the 1's action is the action towards the 1's state. For example if the agent is in the 0's state it will choose the action to move (or better remain) left with a probability of 30% and right with a probability of 70%. If it chooses the 1's action the environment will transition towards the 0's state in 10% of cases and towards the 1's state with 90% of cases. The whole process is the succession of states, actions and rewards. This is the same mechanism, that is more often introduced as the interaction between the agent and the environment. The interaction is nothing more than a Markov chain that is made out of two elements. The first element, the environment is static and can not be changed in any form. The second component is the agent. The agent can change the probabilities of the whole chain, by tweaking the probabilities of actions. In the example above, when the agent starts improving we expect it to increase the probabilities to move to state 0. Ideally that would mean that the agent will only choose action 0.
