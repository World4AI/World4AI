MDP Solution

Once a particular Markov decision process has been defined the next logical step is to solve that MDP. In this chapter we are going to discuss what it actually means to solve the MDP, while the coming chapters are going to focus on how and if we can find a solution.

Return

A return at timestep t undefined, denoted as G_t undefined, is simply the sum of rewards starting from timestep t+1 undefined and going either to terminal state T undefined (episodic tasks) or to infinity (continuing tasks). The letter G undefined stands for Goal, because in reinforcement learning the goal of the environment is encoded in the rewards.

In episodic tasks the return is the sum of rewards in a single episode from time step t+1 undefined to the terminal time step T undefined.

G_t = R_{t+1} + R_{t+2} + … + R_T undefined

In continuing tasks the return is the sum of rewards starting at time step t+1 undefined and going to possibly infinity.

G_t = R_{t+1} + R_{t+2} + R_{t+3} + … = \sum_{k=0}^\infty{R_{k+t+1}} undefined

In the exemplified episodic task below the Markov decision process plays through the sequence of states, actions and rewards all the way to the terminal state T undefined. This sequence of states, actions and rewards of an episode is called a trajectory.

The episode goes from timestep 0 undefined until the terminal timestep T = 8 undefined. For each timestep taken, the agent receives a negative reward of -1 and a reward of +10 when the agent reaches the terminal state S_T undefined.

Based on that trajectory the returns from the perspective of time step 0, 1, and 2 look as follows.

G_0 = R_1 + R_2 + R_3 + R_4 + R_5 + R_6 + R_7 + R_8 \\ = (-1) + (-1) + (-1) + (-1) + (-1) + (-1) + (-1) + 10 = 3 undefined G_1 = R_2 + R_3 + R_4 + R_5 + R_6 + R_7 + R_8 \\ = (-1) + (-1) + (-1) + (-1) + (-1) + (-1) + 10 = 4 undefined G_2 = R_3 + R_4 + R_5 + R_6 + R_7 + R_8 \\ = (-1) + (-1) + (-1) + (-1) + (-1) + 10 = 5 undefined

Discounted Return

Gamma \gamma undefined (also called discount factor) is a value between 0 and 1 (usually between 0.9 and 0.99) that is used to reduce the value of future rewards, because future rewards are considered of less value to the agent than present rewards.

\gamma undefined (gamma) is the discount factor, where 0 \leq \gamma \leq 1 undefined.

Let us consider the following example to convince ourselves that future rewards are indeed of less value than present rewards. If we have to choose between getting 1000$ now and getting 1000$ in 10 years, we should definetely choose the 1000$ in the present. The money could be invested in a 10 year risk free bond, so that at the end of the 10 year period the investor gets back an amout that is larger than 1000$. In reinforcement learning this concept called "time value of money" is extended to all other types of rewards. Specificylly in reinforcement learning we use the geometric series to calculate the current value of a future reward.

S = 1 + \gamma + \gamma^2 + \gamma^3 + \gamma^4 + ... undefined

Therefore the discounted return looks as follows:

G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + ... undefined

Mathematically the discounting is necessary in continuing tasks. If for example at each timestep the agent receives a reward of +1 and the episode has no natural ending, the return G undefined could theoretically become infinite. Due to the geometric progression of gamma (\gamma undefined) the discounted returns approach 0 when they are far into the future.

Policy

A policy is a mapping from states to either actions directly or a probability of an action, depending on whether the policy is deterministic or stochastic.

If a policy is deterministic we define a policy as a mapping from state s undefined to action a undefined. In that case the notation that we use for policy is \mu(s) undefined. An action A_t undefined is generated by using the state S_t undefined as the input into the policy function: A_t = \mu(S_t) undefined.

If a policy is stochastic we define a policy as a mapping from a state s undefined to a probability of an action a undefined, where the mathematical notation is \pi{(a \mid s)} = Pr[A_t = a \mid S_t = s] undefined . This notation can theoretically also be applied to a deterministic case, where policy \pi{(a \mid s) = 1} undefined for for the selected action and \pi{(a \mid s) = 0} undefined for the rest of the legal actions. To generate an action using a stochastic policy we define \pi{(. \mid S_t)} undefined to be the distribution of actions given states, where actions are draws from a policy distribution A_t \sim \pi{(. \mid S_t)} undefined.

Value Functions

In reinforcement learning we often deal with stochastic environments and with stochastic policies. That stochasticity produces different trajectories and therefore different returns G_t undefined even when the starting state S_0 undefined and the policy \pi undefined remain the same. But how can we measure how good it is for the agent to use a certain policy \pi undefined, if the generated returns are not consistent? By using value functions!

For a given policy \pi undefined value functions map states or state-action pairs to “goodness” values, where goodness is calculated as the expected sum of rewards and higher values mean more favorable states or state-action pairs.

In simple words value functions calculate the following: How much return G_t undefined does the agent expect to receive when it faces the state S_t undefined and always follows the policy \pi undefined.

The state-value function v_{\pi}(s) undefined calculates the expected return when the agent is in the state s undefined and follows the policy \pi undefined.

State-Value Function: v_{\pi}(s) = \mathbb{E_{\pi}}[G_t \mid S_t = s] undefined

The action-value function q_{\pi}(s, a) undefined calculates the expected return when the agent faces the state s undefined, takes the action a undefined at first and then keeps following the policy \pi undefined.

Action-Value Function: q_{\pi}(s, a) = \mathbb{E_{\pi}}[G_t \mid S_t = s, A_t = a] undefined

Optimality

At the beginning of the chapter we asked ourselves what it means for an agent to solve a Markov decision process. The solution of the MDP means that the agent has learned an optimal policy function.

For the agent to solve the Markov decision process means to find the optimal policy.

Optimality implies that there is a way to compare different policies and to determine which of the policies is better. In a Markov decision process value functions are used as a metric of the goodness of a policy. The policy \pi undefined is said to be better than the policy \pi undefined if and only if the value function of \pi undefined is larger or equal to the value function of policy \pi' undefined for all states in the state set \mathcal{S} undefined.

\pi \geq \pi’ undefined if and only if v_{\pi}(s) \geq v_{\pi'}(s) undefined for all s \in \mathcal{S} undefined

The optimal policy \pi_* undefined is the policy that is better (or at least not worse) than any other policy.

\pi_* \geq \pi undefined for all \pi undefined

The state-value function and the action-value function that are based on the optimal policy are called optimal state-value and optimal action-value function respectively.

The optimal state-value funtion:

v_*(s) = \max_{\pi} v_{\pi}(s) undefined for all states s \in \mathcal{S} undefined

The optimal action-value function:

q_*(s, a) = \max_{\pi} q_{\pi}(s, a) undefined for all states s \in \mathcal{S} undefined and all actions a \in \mathcal{A} undefined

There might be several optimal policies, but there is always only one optimal value function.

Bellman Equations

An important property of returns is that they can be expressed in terms of future returns to write a more compact definition.

\begin{aligned} G_t & = R_{t+1} + \gamma{R_{t+2}} + \gamma^2{R_{t+3}} + … \\ & = R_{t+1} + \gamma{(R_{t+2} + \gamma{R_{t+3}} + ...)} \\ & = R_{t+1} + \gamma{G_{t+1}} \end{aligned} undefined

By using those properties we can arrive at recursive equations of value functions, where a value of a state can be expressed in terms of future state values.

Bellman equation for the state-value function

\begin{aligned} v_{\pi}(s) & = \mathbb{E_{\pi}}[G_t \mid S_t = s] \\ & = \mathbb{E_{\pi}}[R_{t+1} + \gamma G_{t+1} \mid S_t = s] \\ & = \mathbb{E_{\pi}}[R_{t+1} + \gamma v_{\pi}(S_{t+1}) \mid S_t = s] \end{aligned} undefined

Bellman equation for the action-value function

\begin{aligned} q_{\pi}(s, a) & = \mathbb{E_{\pi}}[G_t \mid S_t = s, A_t = a] \\ & = \mathbb{E_{\pi}}[R_{t+1} + \gamma G_{t+1} \mid S_t = s, A_t = a] \\ & = \mathbb{E_{\pi}}[R_{t+1} + \gamma v_{\pi}(S_{t+1}) \mid S_t = s, A_t = a] \end{aligned} undefined

Equations of the above form are called Bellman equations, named after the mathematician Richard E. Bellman. At the very first glance it might not seem like the equations add additional benefit to the definition of value functions, but the recursive relationships is what makes many of the reinforcement learning algorithms work. Think for example about the return G_t undefined in a continuing environment. The definition of G_t undefined implies that we play for an infinite amount of timesteps before we can calculate a single return. Bellman equations on the other hand will allow us to work with estimates and to use a return from a single timestep to make improvements.

Using the same recursive relationsships we can redefine optimal value functions.

Bellman Optimality Equation for the state-value function:

\begin{aligned} v_*(s) & = \max_{a} q_{{\pi}_*}(s, a) \\ & = \max_{a} \mathbb{E_{\pi_{*}}}[G_t \mid S_t = s, A_t = a] \\ & = \max_{a} \mathbb{E_{\pi_{*}}}[R_{t+1} + \gamma G_{t+1} \mid S_t = s, A_t = a] \\ & = \max_{a} \mathbb{E}[R_{t+1} + \gamma v_*(S_{t+1}) \mid S_t = s, A_t = a] \end{aligned} undefined

Bellman Optimality Equation for the action-value function:

\begin{aligned} q_*(s, a) & = \mathbb{E}[R_{t+1} + \gamma v_*(S_{t+1}) \mid S_t = s, A_t = a] \\ & = \mathbb{E}[R_{t+1} + \gamma \max_{a'} q_*(S_{t+1}, a') \mid S_t = s, A_t = a] \end{aligned} undefined
