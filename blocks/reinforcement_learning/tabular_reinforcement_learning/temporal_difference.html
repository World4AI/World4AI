Temporal Difference Learning

Whereas conventional prediction-learning methods assign credit by means of the difference between predicted and actual outcomes, the new methods assign credit by means of the difference between temporally successive predictions.

Instead of waiting for the end of an episode to use the difference between the predicted state or action value and the actual return G_t undefined temporal difference methods calculate the difference between the value of the current state s undefined and the value of the next state s' undefined.

These methods utilize temporal (as in time) differences in values in their update steps. As we do not have access to actual values for the next state we have to use estimates for target values, we need to use bootstrap. When the agent uses temporal difference methods the agent does not have to wait for the end of the episode to apply an update to the estimate of the value function, but can instead update the estimation function after a single step.

Temporal difference learning is also based on generalized policy iteration, which alternates between policy evaluation and policy improvement to arrive at the optimal policy.

Prediction

To solve the prediction problem means finding the true value function v_\pi(s) undefined for a given policy \pi undefined. We can express this task either using the definition of returns G_t undefined.

v_\pi(s) = \mathbb{E}[G_t \mid S_t = s] undefined

Or we we can use the Bellman equation to rewrite the definition of the value function recursively in terms of the value of the next state.

v_{\pi}(s) = \mathbb{E_{\pi}}[R_{t+1} + \gamma v_{\pi}(S_{t+1}) \mid S_t = s] undefined

Using the same logic the update step can be adjusted to reflect the recursive nature of the value function by using bootstrapping.

\text{MC: }V(S_t) \leftarrow V(S_t) + \alpha [G_t - V(S_t)] undefined {TD: }V(S_t) \leftarrow V(S_t) + \alpha [R_{t+1} + \gamma V(S_{t+1}) - V(S_t)] undefined

Due to bootstrapping the target R_{t+1} + \gamma v_{\pi}(S_{t+1}) undefined can be calculated at each single step. That means that it is not necessary to wait until the episode finishes to improve the estimate and that temporal difference learning is suited for continuing tasks.

V(S_t) \leftarrow V(S_t) + \alpha [\underbrace{R_{t+1} + \overbrace{\gamma V(S_{t+1}) - V(S_t)}^{\boxed{\text{Temporal Difference}}}}_{\boxed{\delta \text{: TD Error}}}] undefined

The quantity V(S_{t+1}) - V(S_t) undefined is the temporal difference: difference of values measured at successive states. The quantity R_{t+1} + \gamma V(S_{t+1}) - V(S_t) undefined is called TD error. TD error the difference between the bootstrapped target and the estimated value. The larger the error \delta undefined, the larger the adjustment that is done to the estimate.

The practical implementation once again relies on a value function contained in a table that maps states s undefined to values V(s) undefined. Unlike in Monte Carlo methods in TD methods the agent updates the value V(s) undefined for the encountered state s undefined after each single step of the interaction.

Compared to Monte Carlo prediction the TD prediction implementation looks cleaner. The interaction step and the update step are consolidated and do not have to be put into different loops or functions.

Control

To find the optimal policy we need to estimate the action value function Q(s,a) undefined. Below we are going to discuss SARSA, the on-policy control algorithm, and Q-learning, the off-policy control algorithm.

SARSA

SARSA is the on-policy TD control algorithm. The same policy that is used to generate actions (\epsilon undefined-greedy in our case) is also the one that is being improved. At each timestep the agent improves the action-value function, by using a bootstrapped version of the target.

Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha [R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t)] undefined

Both actions A_t undefined and A_{t+1} undefined are picked using the \epsilon undefined-greedy policy \pi(a \mid s) undefined.

The name SARSA can be easily understood by writing out those states, actions and the reward that are needed to apply one update step: [S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1}] undefined .

Q-Learning

Q-Learning is the off-policy TD control algorithm. In this algorithm we use \epsilon undefined-greedy to take actions, yet a purely greedy strategy is the one that we aim to improve.

Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha [R_{t+1} + \gamma \max_a Q(S_{t+1}, a) - Q(S_t, A_t)] undefined

Unlike in SARSA, the bootstrapped value from the target is caluclated using the max operator: \max_a Q(S_{t+1},a) undefined.
