Evaluation and Improvement

Similar to dynamic programming and tabular reinforcement learning we are going to apply generalized policy iteration (GPI). In the policy evaluation step we are going to look for a value function \hat{v} undefined that is as close as possible to the true value function v_{\pi} undefined. In the policy improvement step we are going to utilize \hat{q} undefined in order to act greedily and improve our policy.

GPI with an Oracle

We assume that we have some policy \pi undefined and that we are interested in the true value function of that particular policy. Finding the true value function is out of the question du to the continuous observation space, so we have to find an approximation.

\hat{v}(s, \mathbf{w}) \approx v_{\pi}(s) undefined

To build the theory that is going to be used throughout the rest of the book it is convenient to start the discussion by assuming that we are in a supervised learning setting and that there is an oracle that tells us what the true state-value v_{\pi}(s) undefined for the given policy \pi undefined and state s undefined is. Later the discussion can be extended to reinforcement learning settings where the agent interacts with the environment.

In supervised learning the goal is to find a weight vector \mathbf{w} undefined that reduces the error between the true value function v(s) undefined and \hat{v}(s, \mathbf{w}) undefined. In reinforcement learning usually mean squared error (MSE) is used as the measure of error.

MSE \doteq \mathbb{E_{\pi}}[(v_{\pi} - \hat{v}(s, \mathbf{w}))^2] undefined

If we find the weight vector \mathbf{w} undefined that minimizes the above expression, then we found an approximation that is as close as possible to the true value function given by the oracle. The common approach to find such a vector is to use stochastic gradient descent with the following update rule.

\begin {aligned} w_{t+1} & \doteq w_t - \frac{1}{2}\alpha\nabla[v_{\pi}(S_t) - \hat{v}(S_t,\mathbf{w}_t)]^2 \\ & = w_t + \alpha[v_{\pi}(S_t) - \hat{v}(S_t, \mathbf{w}_t)]\nabla\hat{v}(S_t, \mathbf{w}_t) \end {aligned} undefined

Where the gradient \nabla\hat{v}(S_t, \mathbf{w}_t) undefined is a vector that contains partial derivatives of the approximative value function with respect to individual weights.

\large \nabla \hat{v}(s, \mathbf{w}) \doteq \begin{bmatrix} \frac{\partial \hat{v}(\mathbf{w})}{\partial w_1} \\ \frac{\partial \hat{v}(\mathbf{w})}{\partial w_2} \\ \cdots \\ \frac{\partial \hat{v}(\mathbf{w})}{\partial w_n} \end{bmatrix} undefined

To apply the same idea to policy improvement we utilize the action-value function and apply stochastic gradient descent.

\begin {aligned} w_{t+1} & \doteq w_t - \frac{1}{2}\alpha\nabla[q_{\pi}(S_t, A_t) - \hat{q}(S_t, A_t, \mathbf{w}_t)]^2 \\ & = w_t + \alpha[q_{\pi}(S_t, A_t) - \hat{q}(S_t, A_t, \mathbf{w}_t)]\nabla\hat{q}(S_t, A_t, \mathbf{w}_t) \end {aligned} undefined

GPI without an Oracle

The assumption that there is some oracle that gives us the correct state value or action value is obviously not realistic. Therefore we have to interact with the environment to generate the approximate value function. As per usual the choice is between Monte Carlo and TD learning.

With Monte Carlo methods the state value and action value from the oracle is replaced by the return G_t undefined.

\begin {aligned} w_{t+1} & \doteq w_t - \frac{1}{2}\alpha\nabla[G_{t}(S_t) - \hat{v}(S_t,\mathbf{w}_t)]^2 \\ & = w_t + \alpha[G_{t}(S_t) - \hat{v}(S_t, \mathbf{w}_t)]\nabla\hat{v}(S_t, \mathbf{w}_t) \end {aligned} undefined \begin {aligned} w_{t+1} & \doteq w_t - \frac{1}{2}\alpha\nabla[G_t(S_t, A_t) - \hat{q}(S_t, A_t, \mathbf{w}_t)]^2 \\ & = w_t + \alpha[q_{\pi}(S_t, A_t) - \hat{q}(S_t, A_t, \mathbf{w}_t)]\nabla\hat{q}(S_t, A_t, \mathbf{w}_t) \end {aligned} undefined

Temporal Difference methods use bootstrapped values as an approximation for the true state or action values.

\begin {aligned} w_{t+1} & \doteq w_t - \frac{1}{2}\alpha\nabla[R_{t+1} + \hat{v}(S_{t+1},\mathbf{w}_t) - \hat{v}(S_t,\mathbf{w}_t)]^2 \\ & = w_t + \alpha[R_{t+1} + \hat{v}(S_{t+1},\mathbf{w}_t) - \hat{v}(S_t, \mathbf{w}_t)]\nabla\hat{v}(S_t, \mathbf{w}_t) \end {aligned} undefined \begin {aligned} w_{t+1} & \doteq w_t - \frac{1}{2}\alpha\nabla[R_{t+1} + \hat{q}(S_{t+1}, A_{t+1}, \mathbf{w}_t) - \hat{q}(S_t, A_t, \mathbf{w}_t)]^2 \\ & = w_t + \alpha[R_{t+1} + \hat{q}(S_{t+1}, A_{t+1}, \mathbf{w}_t) - \hat{q}(S_t, A_t, \mathbf{w}_t)]\nabla\hat{v}(S_t, \mathbf{w}_t) \end {aligned} undefined
